# labs/

Hands-on exercises, environment configurations, and lab files. This is where you put anything you *do* rather than just read.

## Suggested Structure

```
labs/
└── <subject>/
    └── <lab-name>/
        ├── README.md       # Lab objectives and instructions
        └── <lab-files>     # Config files, manifests, scripts, etc.
```

## Example

```
labs/
└── kubernetes/
    └── deploy-nginx/
        ├── README.md
        └── nginx-deployment.yaml
```
